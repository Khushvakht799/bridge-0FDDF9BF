## 🔭 Обзор технологии
BRIDGE — это не очередной формат данных. Это новая парадигма хранения, передачи и исполнения информации, основанная на принципах семантической плотности, эфемерных процессов и графовой памяти.

- **Семантическая плотность:** BVG упаковывает смысл в 99 байт — в 10-100 раз плотнее JSON/XML
- **Эфемерные процессы:** Процессы живут только пока нужны, схлопываются, оставляя опыт в evolution
- **Графовая память:** Данные помнят свою эволюцию — Git устарел
- **Исполняемый смысл:** BVG-графы могут исполняться в среде Collapse Architecture

## 📦 Компоненты BRIDGE
- **compiler/jvgr_to_bvg.py** — Компилятор JVGr (JSON) → BVG (бинарный граф)
  `python compiler/jvgr_to_bvg.py input.json output.bvg`

- **decompiler/bvg_to_jvgr.py** — Декомпилятор BVG → JVGr (в разработке)
  `python decompiler/bvg_to_jvgr.py input.bvg output.json`

- **utils/bvg_hexdump.py** — Анализатор бинарной структуры BVG
  `python utils/bvg_hexdump.py file.bvg`

## 🔧 Формат BVG (Binary Vector Graph)
```hex
0000: 42 56 47 31 01 00 1c 00 00 00 43 00 00 00 4b 00  BVG1......C...K.
0010: 00 00 c7 dd c1 de 00 00 00 00 00 00 05 00 04 6c  ...............l
0020: 61 6d 70 06 73 65 6e 73 6f 72 0a 63 6f 6e 74 72  amp.sensor.contr
0030: 6f 6c 6c 65 72 08 73 74 72 65 6e 67 74 68 04 70  oller.strength.p
0040: 72 6f 62 02 00 03 00 01 04 00 01 02 00 00 00 00  rob.............
0050: 00 80 3f 33 33 73 3f 01 00 02 00 66 66 66 3f 48  ..?33s?....fff?H
0060: e1 7a 3f                                         .z?
```

BVG — это бинарное представление семантического графа. В 99 байтах упакованы узлы, рёбра, словарь имён и векторы состояний. CRC32 DEC1DDC7 — цифровой отпечаток технологии.

## ⚡ Быстрый старт
```powershell
# Создать JVGr файл (пример в 1_input/)
# Скомпилировать в BVG
python compiler/jvgr_to_bvg.py 1_input/example.json 3_bvg/example.bvg

# Посмотреть содержимое
python utils/bvg_hexdump.py 3_bvg/example.bvg

# Развернуть обратно
python decompiler/bvg_to_jvgr.py 3_bvg/example.bvg 4_output/example.json
```

## 🧠 Четыре принципа
- **Семантической плотности:** Минимальный объем — максимальный смысл
- **Схлопывания:** Все временное исчезает, оставляя только опыт
- **Графовой памяти:** Связи важнее узлов, эволюция важнее состояний
- **Исполняемого смысла:** Данные должны уметь действовать, а не только храниться
